# Безопасность и персональные данные

Этот документ фиксирует технические и продуктовые требования, которые нужно учитывать при переходе от MVP к рабочей платформе.

## 1. Важное ограничение текущего MVP

Сейчас приложение не имеет полноценного backend, авторизации, серверной проверки прав и защищенного хранения данных. Поэтому текущий вариант подходит для прототипа и демонстрации, но не для хранения реальных детских данных в продакшене.

## 2. Что нужно добавить

- авторизацию;
- роли;
- серверную проверку доступа;
- защищенные приглашения;
- журнал действий;
- хранение согласий;
- возможность удалить данные ученика;
- резервные копии;
- разграничение данных между студиями;
- хранение данных в России.

## 3. Видео учеников

Желательный безопасный подход:

- не хранить видео ученика по умолчанию;
- анализировать видео на устройстве;
- сохранять результат и точки скелета;
- видео сохранять только при явном согласии;
- предусмотреть срок хранения;
- предусмотреть удаление видео.

## 4. Родительское согласие

Если ученик несовершеннолетний, регистрация должна включать шаг родителя или законного представителя:

- подтверждение личности представителя;
- согласие на создание учебного профиля;
- согласие на обработку персональных данных;
- отдельное согласие на обработку видео, если видео будет храниться;
- фиксация даты, IP, user-agent и версии документа.

## 5. Права доступа

Ученик не должен иметь возможность открыть чужие уроки, чужую группу или чужую студию через изменение URL или localStorage.

Все права доступа должны проверяться на сервере.
