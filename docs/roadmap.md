# Roadmap

## Этап 0. Стабилизация текущего MVP

- Зафиксировать текущий рабочий UI.
- Не смешивать дизайн-правки с архитектурными изменениями.
- Проверить незакоммиченные и временные файлы.
- Поддерживать Cloudflare Pages deploy через GitHub.

## Этап 1. Документация и структура

- Поддерживать `docs/full-migration-spec.md`.
- Описать продуктовую структуру.
- Описать модель данных.
- Описать безопасность и юридические точки.
- Создать каркас будущей структуры кода.

## Этап 2. Аккуратный frontend-рефакторинг

- Выделить storage.
- Выделить scoring.
- Выделить MediaPipe.
- Выделить UI-компоненты.
- Выделить кабинеты.
- Разобрать CSS.

## Этап 3. Backend MVP

- Поднять backend API.
- Подключить PostgreSQL.
- Подключить объектное хранилище.
- Реализовать студии, группы, уроки, учеников.
- Реализовать авторизацию и роли.

## Этап 4. Реальные данные

- Перевести студии и уроки из localStorage на backend.
- Перевести учеников и попытки на backend.
- Реализовать invite-ссылки и QR-коды через сервер.
- Реализовать права доступа.

## Этап 5. Юридический и безопасный контур

- Согласия родителей.
- Журнал действий.
- Политика удаления данных.
- Резервные копии.
- Хранение данных в России.

## Этап 6. Масштабирование

- Тарифы.
- Платежи.
- Панель платформы.
- Мониторинг.
- Аналитика по студиям.
- Расширенное тестирование.
