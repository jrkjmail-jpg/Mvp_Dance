# MVP backlog

Документ фиксирует, что еще нужно доделать, переделать или проверить в текущем MVP перед переходом к полноценной платформе.

Это не список задач на немедленную разработку. Это рабочая доска направлений.

## 1. Интерфейс и дизайн

### Шапка

- окончательно зафиксировать логотип “Танцуй”;
- проверить desktop, iPhone, iPad;
- проверить выдвигающееся меню аккаунта;
- убрать все лишние аватары рядом с логотипом;
- сохранить единый вид бренда во всех кабинетах.

### Кабинет ученика

- проверить мобильную карточку ученика;
- проверить карточку уровня;
- проверить историю уроков и любимые;
- проверить отображение студии ученика;
- проверить пустые состояния;
- проверить, что аватар не прячется за блоками.

### Кабинет педагога

- проверить страницу студии;
- проверить кнопку “Пригласить”;
- проверить создание групп и подгрупп;
- проверить карточки групп;
- проверить карточки уроков;
- проверить удаление уроков/групп/подгрупп;
- проверить загрузку обложки урока.

### Пространство студии

- единый вид для педагога и ученика;
- ученик видит только доступные группы и уроки;
- пустые состояния выглядят одинаково;
- группы и подгруппы аккуратно помещают любой короткий текст.

## 2. Логика уроков

- создание урока;
- загрузка видео обучения;
- загрузка видео эталона;
- выставление начала и конца;
- публикация урока;
- отображение урока в группе;
- доступ ученика к уроку;
- просмотр урока учеником;
- сдача урока;
- результат и звезды.

## 3. Видео и MediaPipe

- проверить, что скелет не отстает на iPhone;
- проверить загруженное видео ученика;
- проверить камеру ученика;
- проверить видео педагога;
- оставить стабильную модель без heavy/3D/smoothing, если они ухудшают работу;
- отдельно исследовать, как поднять FPS без задержки скелета.

## 4. Данные

Сейчас данные хранятся локально. Для MVP нужно решить, какие данные пока остаются локальными, а какие первыми уйдут на backend.

Минимальный набор будущих серверных данных:

- студии;
- группы;
- ученики;
- уроки;
- попытки;
- результаты;
- приглашения;
- согласия.

## 5. Регистрация и приглашения

- приглашение ученика через ссылку/QR;
- родительский шаг регистрации;
- анкета ученика;
- согласия;
- статус “ждет регистрации”;
- перевод ученика в группу после регистрации.

## 6. Безопасность

- убрать доверие к localStorage для прав доступа;
- добавить серверные роли;
- добавить серверную проверку доступа к урокам;
- защитить invite-ссылки;
- добавить журнал действий;
- продумать хранение детских данных.

## 7. Что не трогать без отдельного решения

- не менять MediaPipe-модель без теста на iPhone;
- не добавлять heavy/3D обратно без отдельной ветки;
- не переписывать всю верстку за один раз;
- не смешивать дизайн-правки и рефакторинг;
- не делать backend до согласования модели данных.
