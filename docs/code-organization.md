# План аккуратного упорядочивания кода

Этот документ фиксирует правила, по которым проект нужно постепенно переводить из текущего монолитного MVP в более устойчивую структуру.

Главный принцип: сначала создаем каркас и границы ответственности, потом переносим код маленькими частями. Нельзя переносить большие куски `app.js` и `styles.css` одним рывком.

## 1. Что нельзя делать резко

- Не разбивать весь `app.js` за один коммит.
- Не переподключать сразу много новых файлов в `index.html`.
- Не переносить CSS без визуальной проверки desktop, iPhone и iPad.
- Не менять MediaPipe и scoring одновременно с UI.
- Не смешивать рефакторинг и новую функциональность в одном коммите.

## 2. Безопасный порядок

### Шаг 1. Документация и структура

Создать папки и описать назначение модулей. Код приложения пока остается на месте.

### Шаг 2. Выделить чистые функции

Первыми можно выносить функции, которые не зависят от DOM:

- расчет звезд;
- нормализация данных;
- форматирование результата;
- работа с localStorage;
- утилиты для дат, текста и файлов.

### Шаг 3. Выделить MediaPipe

После чистых функций можно выделять:

- загрузку MediaPipe;
- создание landmarker;
- чтение кадров;
- отрисовку скелета.

Этот шаг должен быть отдельным, потому что MediaPipe критичен для производительности на телефонах.

### Шаг 4. Выделить экраны

После этого можно постепенно разделять:

- кабинет ученика;
- кабинет педагога;
- страницу студии;
- пространство материалов;
- редактор урока.

### Шаг 5. Разобрать CSS

CSS переносить только после того, как станет понятно, какие UI-блоки являются повторяемыми компонентами.

## 3. Правило одного изменения

Каждый коммит должен отвечать на один вопрос:

- либо “я перенес storage”;
- либо “я перенес scoring”;
- либо “я перенес кнопки”;
- либо “я исправил мобильную шапку”.

Не смешивать.

## 4. Минимальная проверка после каждого шага

После каждого переноса проверять:

- приложение открывается;
- шапка не съехала;
- переключение кабинет ученика / педагог работает;
- студия открывается;
- урок открывается;
- мобильная версия не ломается;
- консоль браузера без новых ошибок.

## 5. Приоритеты выноса из `app.js`

1. `storage` — localStorage и сериализация данных.
2. `scoring` — расчет результата и звезд.
3. `pose` — MediaPipe и скелет.
4. `media` — видео, камера, обложки.
5. `ui` — повторяемые компоненты.
6. `student`, `teacher`, `studio` — экраны и сценарии.

## 6. Приоритеты выноса из `styles.css`

1. переменные и токены;
2. кнопки;
3. карточки;
4. модальные окна;
5. шапка;
6. видео-блок;
7. формы;
8. экраны;
9. адаптив.
